the setup
You built one decision tree in the last activity. The problem: a single tree is brittle — change a few training parcels and the whole structure can shift.

A random forest fixes this by building many trees, each on a different random subsample of the data and a random subset of features. No tree sees everything. Then they vote.

Below are 3 trees trained on overlapping but different subsets of the 15 Houston parcels. Notice they each chose a different first split — and they'll disagree on some borderline cases. That disagreement is the point.
key concept
Why does using less data per tree make the forest more accurate?

Each tree makes different errors because it saw different training data. When you aggregate many imperfect trees, errors that are idiosyncratic to one tree get outvoted. Only errors that are systematic across all trees survive — and those reflect genuine difficulty in the data, not noise.
In real RF: sklearn builds 100–500 trees. Here we use 3 so you can read them. The principle is identical.
