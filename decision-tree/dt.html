Flood risk prediction
15 parcels, 4 features. You choose every split — the tree builds as you decide.
the scenario
A county planning office wants to predict which parcels are at HIGH flood risk, using only features they can measure from GIS layers — no historical insurance data needed.

You have 15 labeled parcels (past flood events confirmed the risk), each described by 4 features:
dist. to stream — proximity to nearest waterway (m)
imp. surface % — % of parcel covered by pavement/rooftop
slope — average slope in degrees
drainage class — soil drainage capacity (poor / mod / good)
how a decision tree works
A decision tree is a series of if/then rules learned from data. Starting from the full dataset, it repeatedly asks: "which feature and threshold best separates the two classes?"
1 Pick the feature + threshold that creates the purest split — one side mostly YES, one side mostly NO.
2 Repeat on each branch until a branch is pure enough to stop — that branch becomes a leaf node with a final prediction.
3 Label each leaf by majority vote — whichever class appears most in that leaf becomes the prediction.
Key metric: Gini impurity. 0 = perfectly pure (all one class). 0.5 = maximally mixed (50/50 split). You want splits that drive both sides toward 0.

Anatomy of a decision tree:
Decision node — asks a yes/no question. All parcels flow in, split into two groups based on the answer.
Leaf node (prediction) — the end of a branch. No more splitting. Majority class = the tree's prediction for any parcel that lands here.
Gini impurity — shown at each node. Dropping from 0.48 → 0.18 means this split substantially separated the classes.
